## Overview

SIVAKUMAR S supports seamless integrations with popular external services. You can set up API keys for programmatic access, configure webhooks for real-time notifications, connect to Git for version control, and link collaboration apps like Slack or Microsoft Teams. These integrations streamline your documentation workflow and keep your team in sync.

<Callout kind="info">
  Before integrating, generate an API key from your dashboard at `https://dashboard.example.com/settings/api-keys`.
</Callout>

## Setting Up API Keys

API keys allow external services to authenticate requests to SIVAKUMAR S. Follow these steps to create and use one.

<Steps>
  <Step title="Generate Key" icon="key">
    Log in to `https://dashboard.example.com`, navigate to Settings > API Keys, and click "Create New Key".
  </Step>
  <Step title="Copy Key" icon="copy">
    Copy the generated key securely. It displays only once.
  </Step>
  <Step title="Test Key" icon="play">
    Use the key in a test request:

````javascript
const response = await fetch('https://api.example.com/v1/docs', {
  headers: {
    'Authorization': `Bearer YOUR_API_KEY`
  }
});
````

  </Step>
</Steps>

<ParamField header="Authorization" param-type="string" required="true">
  Format: `Bearer YOUR_API_KEY`
</ParamField>

## Configuring Webhooks

Webhooks notify external services about events like document updates. Configure them via the dashboard or API.

<CodeGroup tabs="Dashboard,API">
  ```json
  {
    "url": "https://your-webhook-url.com/webhook",
    "events": ["doc.updated", "doc.published"]
  }
  ```
  ```bash
  curl -X POST https://api.example.com/v1/webhooks \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -d '{"url": "https://your-webhook-url.com/webhook", "events": ["doc.updated"]}'
  ```
</CodeGroup>

<Callout kind="tip">
  Validate webhook payloads using HMAC signatures for security.
</Callout>

## Git Integration

Connect SIVAKUMAR S to GitHub or GitLab to sync documentation changes bidirectionally.

### Setup Steps

1. Authorize SIVAKUMAR S in your Git provider's app settings.
2. Select repositories in `https://dashboard.example.com/integrations/git`.
3. Push changes from Git to trigger auto-publishes.

```yaml
# .github/workflows/docs.yml
name: Sync Docs
on: push
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Sync to SIVAKUMAR S
        run: curl -X POST https://api.example.com/v1/repos/sync \
          -H "Authorization: Bearer ${{ secrets.SIVAKUMAR_S_KEY }}"
```

## Collaboration Apps

Integrate with team communication tools using webhooks or direct apps.

<Tabs>
  <Tab title="Slack" icon="message-circle">
    Add the SIVAKUMAR S app to your workspace or use incoming webhooks.

````javascript
// Slack webhook payload
{
  "text": "Document *Quickstart* updated by @user",
  "attachments": [{
    "title": "View Changes",
    "title_link": "https://docs.example.com/quickstart"
  }]
}
````

  </Tab>
  <Tab title="Microsoft Teams" icon="microsoft-teams">
    Create an incoming webhook in Teams channel settings.

````json
{
  "type": "message",
  "attachments": [{
    "contentType": "Application/Vnd.Microsoft.Card Adaptive",
    "content": {
      "type": "AdaptiveCard",
      "body": [{
        "type": "TextBlock",
        "text": "New doc update in SIVAKUMAR S"
      }]
    }
  }]
}
````

  </Tab>
</Tabs>

<Expandable title="Advanced GitHub Actions" default-open="false">

For complex workflows, use GitHub Actions with SIVAKUMAR S API.

```yaml
- name: Deploy Preview
  run: |
    curl -X POST https://api.example.com/v1/previews \
      -H "Authorization: Bearer ${{ secrets.API_KEY }}" \
      -d '{"branch": "${{ github.ref_name }}", "docs": ["introduction.mdx"]}'
```

</Expandable>

## Best Practices

- Rotate API keys regularly.
- Use least-privilege scopes.
- Monitor webhook delivery rates in the dashboard.
